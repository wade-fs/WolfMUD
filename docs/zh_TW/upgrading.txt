
名稱

  upgrading - 關於從一個版本的 WolfMUD 升級到下一版本的一些注意事項

說明

  ++ 請在升級您的 WolfMUD 服務器之前閱讀本文檔的全部內容。 ++

  本文檔描述了將 WolfMUD 升級到新版本時要採取的步驟，以免丟失播放器文件、自定義或修改的區域文件或配置更改。

  重要的目錄是包含服務器配置、區域文件和播放器文件的數據目錄。

  備份它或將數據目錄複製到安全的地方很重要。對於大多數安裝，只需將 WolfMUD 目錄重命名為 WolfMUD.old 就足以備份所有內容。

  安裝新版本的 WolfMUD 後，您可以刪除默認數據目錄或重命名以供以後參考，並將其替換為您對自己的數據目錄製作的備份或副本。

  如果有任何文件需要升級，例如配置文件中的新設置或播放器或區域文件中的新字段，服務器將在服務器啟動時檢查文件並根據需要進行升級。

  此外，您可能希望使用文件比較工具將默認配置文件和/或區域文件與您的版本進行比較以進行更改。

WOLFMUD 目錄

  在 v0.0.18 之前的 WolfMUD 版本中，主要的 WolfMUD 目錄應該是 WolfMUD.git 用於源下載或存儲庫的 Git 克隆。 從 v0.0.18 開始，該目錄將為 WolfMUD。 升級後建議您將 WolfMUD.git 目錄重命名為 WolfMUD 以方便以後的更新。

環境變數

  在 v0.0.18 之前的 WolfMUD 版本中，使用源下載或 Git 存儲庫的克隆時，必須設置環境變量 WOLFMUD_DIR。 對於標準默認安裝，這不再是必需的。 WOLFMUD_DIR 只需要為自定義配置設置。

  有關 WOLFMUD_DIR 的詳細信息，請參閱文檔目錄中文件 running-the-server.txt 中的環境變量部分。
